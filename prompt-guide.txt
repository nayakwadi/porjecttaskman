1. Ask the right questions to get right response
2. Be very specific about what we are expecting and keep requirements, functionalities, output formats, with precise scenarios
3. Try to breakdown the application into modules and provide more specific instructions about using specific libraries, methods, approaches etc.
4. Provide the Context: Who you are, what are you trying to do, how do you want the output formats, adding files for referrence, possible outcomes with some examples
5. The more precise prompt would give more specific output in the very first attempt. Iterate the prompts as you make changes, try out new things.
6. Ultimately all prompts are being converted into tokens (LLMs breakdown incoming prompts into tokens, works on those tokens, gathers outputs, formats the outcome)
7. Ensure to create the prompt with proper structure i.e. Split the prompt into sections, be specific about each sections
8. Iterate and refine the prompt. 
9. the transcript generated can be used as main prompts.md file and keep modifying the prompts for a better outcome
10. Try exploring modes like 
    a. Ask (eg: To undesratnd a code block in a specific file)
    b. Plan (eg:Create a step-by step plan to build an application x with requirements abc and targeted users who expects xyz functionalities)
    c. Agent (eg: Generate Code based on the prompts provided with referring to context or no examples at all)
11. Refer to Five Pillar Framework mentioned in this file prompt-5-pillars.md
12. Avoid single line prompts which do not yield expected outcomes and it leads to end less back-and-forth iterations. 
    Also it might just  work in isolation and conflict during integration



Examples:
Worst Prompt: Build a task management application using basic Flask, python, html, SQLLite

In the above example , we provided a prompt which is wide open and things would go out of control as we are alloing AI to be more creative.
We cannot expect a working application as an outcome and ultimately we may end up rewriting our prompts until we see some valid result.


Better Prompt: Constructing a huge prompt with pieces, each piece has specific instructions/examples/suggestions 
Context, Objective, Functionalities, Requirements, Application Set up etc. Refer to the file /cursor_prompts_taskmanager_app.md



Avoid Zero Shot prompts  i.e. Not providing examples
Always use atleast One shot or Multi shot examples.
One Shot example: Write a regular expression that matches valid email addresses. Ensure to test it with below examples
- valid@example.com
- invalid@
- myusername.com 
- @nodomain.com

Few Shot or Multi Shot example:
Create a Day to day Task Planner application which can be run in local.It should have all the capabilities mentioned in beloe exampels
Example 1: Outlook task management
Example 2: IPhone reminders
Example 3: Google Calendars


Also consider mentioning Chain-of-thought approach, like providing hints in the prompt for think through approach
Chain-of-thought example: Design a database schema for a task management application. Think through each entity, relationships, constraints step by step.
For bigger projects, try to include those necessary project folders/files and refer those in the context

You can also request to provide multiple outputs and pick one suitable for your needs.
example: Write a regular expression that validates email addresses. Create 4 examples and evaluate pros and cons of each. Pick the best method with atmost efficiency 
and can be used for a large scale application.
